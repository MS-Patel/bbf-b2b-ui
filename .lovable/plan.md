
## Investor Onboarding Workflow — UI-only Plan

Build a production-grade 9-step Investor Onboarding workflow as a self-contained module that plugs into the existing B2B app shell. UI only — no API calls, no Zustand store, no React Query. Local component state + plain TypeScript types. Reuses the existing `src/components/admin/*` library (`StepForm`, `UploadZone`, `Timeline`, `KPIWidget`, `StatusBadge`, etc.) so the look matches the rest of the platform.

Note on routing: the project uses TanStack file-based routing. The task says "no routeTree.gen.ts" — we will NOT hand-edit `routeTree.gen.ts` (it is auto-generated by the Vite plugin), and we will not introduce a different router. We add ONE route file `src/routes/app.admin.onboarding.investor.tsx` that hosts `InvestorOnboardingPage`. All real work lives in plain components under `src/features/investor-onboarding/` so it can be lifted into any architecture later.

### Step-by-step UI specs

1. **Identity** — PAN input with `AsyncFieldStatus` (idle/checking/verified/duplicate-found). On "duplicate found", inline warning card with link "View existing investor". Radio group for investor type (Individual, Joint, Minor, NRI, Corporate). Conditional sub-sections: Guardian (Minor), Joint Holders repeater (Joint), Incorporation date (Corporate).

2. **KYC & Compliance** — Authoritative "NDML Verification" card with large status badge (Pending/Verified/Rejected/Expired), provider reference, last-checked timestamp, "Re-run verification" button. Below it a `Timeline` of verification events. Warning panel if expired/rejected with remediation guidance.

3. **FATCA & Taxation** — "Tax residency" repeater (Country, TIN, TIN-not-available reason). Declaration checkboxes. Classification select (Individual / Non-Financial Entity / Financial Institution). Collapsible "Advanced compliance" accordion for CRS specifics — extensible.

4. **Bank Accounts** — Grid of bank account cards (bank name, masked account, IFSC, primary star, status pill). "Add account" opens an inline drawer with IFSC `AsyncFieldStatus` lookup that fills bank+branch. Set-primary, edit, deactivate actions. Empty state when none.

5. **Nominees** — Up to 3 nominee rows with name, relation, DOB, % allocation. `AllocationMeter` shows stacked bar of allocations + remaining; turns red if total ≠ 100. Minor nominee triggers Guardian sub-fields. Inline validation summary at top.

6. **Risk Profile** — Grouped questionnaire (Investment horizon / Loss tolerance / Experience / Goals). Live computed score in a side panel with classification (Conservative/Moderate/Aggressive), suitability statement, and "Override classification" with mandatory note + warning banner.

7. **Relationship Mapping** — Searchable selectors for RM and Distributor (combobox), branch/franchise auto-derived as read-only context cards. Hierarchy summary panel: Franchise → Branch → Distributor → RM → Investor (vertical chain with avatars/initials).

8. **Documents** — Required documents grid using `UploadZone` per doc (PAN, Cancelled Cheque, Address Proof) + optional docs section (FATCA form, Signature). Each tile shows file name, size, upload progress, and status pill (Uploaded/Verifying/Verified/Rejected). Replace/remove actions.

9. **Review & Submit** — KPI strip (sections complete, validation issues, documents uploaded). Eight collapsible `ReviewSectionCard`s (one per prior step) showing key fields + per-field status icons and a "Jump to section" link. Bottom: declaration checkboxes, consent panel (data processing, terms, marketing opt-in), and primary "Submit onboarding" CTA disabled until all required checks pass.

---

### Draft / unsaved-changes UX (UI only)

- `useDraftState` local hook (inside the page) keeps form state + `isDirty` + `lastSavedAt` in `useState`. No persistence, no API — "Save draft" just sets `lastSavedAt = new Date()` and shows a toast. This is purely a UX shell so the eventual backend integration can wire in.
- `UnsavedChangesDialog` blocks step navigation when current step is dirty.

### Out of scope (explicit)

- No API client, server functions, Zustand store, or React Query.
- No real PAN/IFSC lookup — `AsyncFieldStatus` is driven by local timeouts on a mock map in `fixtures.ts`.
- No auth/role guard changes beyond mirroring the existing admin route guard pattern.
- No edits to `routeTree.gen.ts`, no design-token changes, no new colors.

### Technical notes

- All types live in `types.ts`; all option lists / step metadata in `constants.ts`. Components consume props — no hidden globals.
- Step components share a common `StepProps<T>` shape: `{ value, onChange, errors, onValidate }` so they can be lifted into any host form later.
- File names use kebab-case to match project convention; routes follow TanStack flat-dot naming.
- Strictly Tailwind + shadcn primitives + existing admin components — no new dependencies.
